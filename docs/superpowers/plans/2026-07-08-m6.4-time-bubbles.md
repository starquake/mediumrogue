# Milestone 6.4 — Time Bubbles Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or executing-plans. Steps use `- [ ]` checkboxes.
> **DRAFT for review — do not build until approved.** Largest slice of M6; builds on 6.3 (rebase onto `main` after #16 merges).

**Goal:** Local, action-gated combat time — bubbles freeze and run on their own clock (all-players-ready or patience timeout) while the world keeps ticking; walk in to join, break away to leave.

**Architecture:** Entities carry a domain (`bubbleID`: 0 = world, else a bubble). Bubbles are recomputed as connected components of a `HexDistance <= CombatRadius` graph that contain an opposing pair. The world domain resolves on `TURN_INTERVAL` (6.1/6.3 phased move); each bubble resolves the 6.3 combat turn when its players lock in or its patience timer fires. One control loop drives both.

## Global Constraints
- **Distance-based** detection (hex distance ≤ `CombatRadius=6`); terrain-blocked LOS is deferred.
- Reuse the 6.3 combat resolution (`moveAndBumpLocked`/`attackLocked`/`resolveDeathsLocked`) unchanged — 6.4 scopes *which entities* resolve together and *when*, not how a combat turn works.
- Every entity is in exactly one domain; bubbles form/grow/merge/dissolve purely from the connected-component recompute.
- All state transitions hold `w.mu`; no double-resolution of a bubble-turn.
- New timing knobs env-configurable and shrinkable in tests: `COMBAT_PATIENCE` (default 60s), `BUBBLE_POLL` (default 100ms) — thread like `TURN_INTERVAL`.
- `internal/protocol` is the wire source of truth; `make protocol` after edits. `got, want` tests; `make check` + `make e2e` green. One PR off `main` (branch `m6.4-time-bubbles`).

## Task 2: Game — domains & bubble formation (pure recompute)

**Files:** create `internal/game/bubble.go`; modify `internal/game/world.go`; tests in `internal/game/bubble_test.go`.

Establish the domain model as a **pure function of positions** — no clock changes yet. After this task, bubbles form/merge/dissolve/absorb correctly and `InCombat` is on the wire; resolution still happens on the world clock (temporary, until Task 3).

- [ ] **entity + world state.** Add `bubbleID int64` to `entity` (0 = world domain). Add to `World`: `bubbles map[int64]*bubble`, `nextBubbleID int64`. Define in `bubble.go`:

```go
// bubble is a set of entities frozen in local combat time, resolving on its own
// action-gated clock (see resolveBubbleLocked). Rebuilt each recompute; id is
// carried across recomputes by membership overlap so gating state is stable.
type bubble struct {
	id       int64
	members  map[int64]struct{} // entity ids
	ready    map[int64]struct{} // player ids locked in this bubble-turn
	deadline time.Time          // patience timeout for the current bubble-turn
}
```

- [ ] **`recomputeBubblesLocked`.** Connected components over `HexDistance(a,b) <= CombatRadius`; a component is a bubble iff it contains ≥1 opposing pair (a `player` and a `monster`). Assign each member's `bubbleID`; world-domain entities get `bubbleID = 0`. Carry bubble ids across recomputes by **max membership overlap** with the previous `w.bubbles` (so `ready`/`deadline` survive); new components get `nextBubbleID++`; merges keep the largest-overlap id. Sketch:

```go
func (w *World) recomputeBubblesLocked() {
	comps := connectedComponents(w.entitiesSlice(), protocol.CombatRadius) // [][]*entity
	next := make(map[int64]*bubble)
	for _, comp := range comps {
		if !hasOpposingPair(comp) {
			for _, e := range comp { e.bubbleID = 0 }
			continue
		}
		id := w.inheritBubbleID(comp)         // max-overlap with w.bubbles, else nextBubbleID++
		b := w.carryOrNewBubble(id, comp)     // preserve ready/deadline if inherited
		for _, e := range comp { e.bubbleID = id; b.members[e.id] = struct{}{} }
		next[id] = b
	}
	w.bubbles = next
}
```
(`connectedComponents`, `hasOpposingPair`, `inheritBubbleID`, `carryOrNewBubble` are small helpers authored here.)

- [ ] **Snapshot.** Set `InCombat: e.bubbleID != 0` per entity; build `Bubbles` from `w.bubbles` (member ids sorted; `WaitingForIDs` = members that are players and not in `ready`; `PatienceRemainingMs` from `deadline` — 0 until Task 3 wires the clock).

## Task 3: Game — the two clocks & action-gating (the core)

**Files:** `internal/game/world.go` (+ `bubble.go`); tests in `internal/game/bubble_test.go` / a new `combat_gating_test.go`.

- [ ] **Split resolution.**
  - `resolveWorldLocked()` — the current phased move/combat, but iterating **world-domain entities only** (`bubbleID == 0`); world monsters target the nearest **world** player. Ends by `recomputeBubblesLocked()`.
  - `resolveBubbleLocked(b *bubble)` — run the 6.3 combat turn (`moveAndBumpLocked`+`attackLocked`+`resolveDeathsLocked`) over **`b`'s members only**; reset `b.ready`; then `recomputeBubblesLocked()` and restart the patience deadline for surviving bubbles.
  - Factor the shared move/attack code so both call it scoped to an entity set (pass the member slice instead of ranging all `w.entities`).
- [ ] **Lock-in.** `SubmitIntent` for an entity whose `bubbleID != 0` records `b.ready[entityID] = struct{}{}` (in addition to setting the path). If, after that, **all member players are ready**, `resolveBubbleLocked(b)` immediately (then publish). A guard (per-bubble turn counter or a `resolving` flag) prevents the poll path from re-resolving the same bubble-turn.
- [ ] **Control loop.** Replace `Run`'s single ticker with one loop on `BubblePoll` cadence:

```go
func (w *World) Run(ctx context.Context) {
	poll := time.NewTicker(w.bubblePoll)
	defer poll.Stop()
	last := time.Now() // world-tick accounting; injectable clock for tests
	for {
		select {
		case <-ctx.Done(): return
		case now := <-poll.C:
			changed := false
			w.mu.Lock()
			if now.Sub(last) >= w.interval { w.resolveWorldLocked(); last = now; changed = true }
			for _, b := range w.bubbles {
				if w.bubbleReadyOrExpiredLocked(b, now) { w.resolveBubbleLocked(b); changed = true }
			}
			w.mu.Unlock()
			if changed { w.ticks.Publish() }
		}
	}
}
```
`bubbleReadyOrExpiredLocked` = all member players ready **or** `now.After(b.deadline)`. New bubbles get `deadline = now.Add(CombatPatience)` on formation.
- [ ] **Patience remaining** on the wire: `PatienceRemainingMs = max(0, deadline - now)` in `Snapshot`.
- [ ] **Tests**: a formed bubble does **not** advance while the player is unready (world `turn` advances, bubble members don't move); it advances once the player locks in (combat turn runs, HP changes); a tiny `COMBAT_PATIENCE` advances it with an unready player waiting; the world domain keeps resolving independently; escape/dissolve still hold. Use an injectable clock or a `ResolveBubbleForTest`/`SetDeadlineForTest` bridge for determinism.
- [ ] Verify + `make lint`; commit.

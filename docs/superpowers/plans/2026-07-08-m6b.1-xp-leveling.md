# Milestone 6b.1 — XP & Leveling Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or executing-plans. Steps use `- [ ]` checkboxes.

**Goal:** Players earn shared XP from kills, gain (derived) levels, and lose within-level progress on death — completing the milestone-6 "no XP penalty yet" debt.

**Architecture:** `Entity` carries `XP`/`Level` on the wire (level derived from XP). XP is awarded in `resolveDeathsLocked` — the death-resolution step already receives the fight's member set, so on each monster death every surviving **player** member gains `MonsterXP` (full, per kill). Player death floors XP to the current level's start.

## Global Constraints
- Level is **derived** from XP (`level = 1 + xp/XPPerLevel`), never stored independently — one source of truth via a `levelFor(xp)` helper reused by the penalty, the wire, and (client) the HUD.
- XP recipients = the killed monster's **fight** = the `members` set passed to `resolveDeathsLocked` (a bubble's members, since all combat happens in bubbles); each surviving player gets the FULL `MonsterXP` per kill (not divided).
- Death floors XP to `(xp/XPPerLevel)*XPPerLevel` — keeps the level, loses within-level progress. Respawn otherwise unchanged (HP/pos/path, same id+token — 6.3).
- Constants in `internal/protocol` (`XPPerLevel`, `MonsterXP`); `make protocol` after wire edits. `got, want` tests; `make check` + `make e2e` green. Monsters have no XP (fields stay 0). One PR off `main` (branch `m6b.1-xp-leveling`).
- No level-up mechanical effects yet (that's classes/species, 6b.2/6b.3) — this slice only tracks level.

## Task 2: Game — XP award, level, death floor

```go
// levelFor returns the 1-based level for a cumulative XP total.
func levelFor(xp int) int { return 1 + xp/protocol.XPPerLevel }

// levelFloorXP returns the XP at the start of xp's current level.
func levelFloorXP(xp int) int { return (xp / protocol.XPPerLevel) * protocol.XPPerLevel }
```

- [ ] **Snapshot** carries XP/Level: `XP: e.xp, Level: levelFor(e.xp)` per entity (monsters have `xp==0` → `Level 1`; that's fine, clients only read their own).
- [ ] **Award + floor in `resolveDeathsLocked`.** Modify it: count dead monsters in `members`; award `killed*MonsterXP` to each surviving player member; floor XP on player death. New body:

```go
func (w *World) resolveDeathsLocked(members []*entity) {
	var dead []*entity

	monstersKilled := 0

	for _, e := range members {
		if e.hp <= 0 {
			dead = append(dead, e)

			if e.kind == protocol.EntityMonster {
				monstersKilled++
			}
		}
	}

	// Shared XP: every player who survived this fight gets the full MonsterXP for
	// each monster that fell — no last-hit competition, helping always pays.
	if monstersKilled > 0 {
		for _, e := range members {
			if e.kind == protocol.EntityPlayer && e.hp > 0 {
				e.xp += monstersKilled * protocol.MonsterXP
			}
		}
	}

	slices.SortFunc(dead, func(a, b *entity) int { return int(a.id - b.id) })

	for _, e := range dead {
		if e.kind == protocol.EntityMonster {
			delete(w.entities, e.id)

			continue
		}

		// Player: respawn (same id+token), and fall back to the start of the XP
		// level you were in — keep the level, lose the within-level progress.
		e.xp = levelFloorXP(e.xp)

		if spawn, err := w.spawnHexLocked(); err == nil {
			e.hex = spawn
		}

		e.hp = e.maxHP
		e.path = nil
	}
}
```

# Milestone 6b.2 — Classes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or executing-plans. Steps use `- [ ]` checkboxes.

**Goal:** Three playable classes (fighter/rogue/mage) with distinct combat — per-class stats + level scaling, class-default equipped weapons, melee bump damage by weapon, and a new ranged attack intent (rogue bow single-target, mage AoE).

**Architecture:** `Entity.Class`; each entity has class-default close/ranged weapons. A `statsFor(class, level)` + `weaponDamage(weapon, level)` are the single source of truth for HP/damage. Melee stays bump-to-attack (now class-weapon damage); a new `AttackIntent{target}` drives ranged/AoE, resolved in the bubble attack phase alongside bumps.

## Global Constraints
- Manual bump (melee) + explicit ranged **attack intent** (rogue bow, mage AoE); fighter has no ranged. No manual weapon swapping (weapon = how you engage). No friendly fire (ranged hits opposing faction only). Ranged resolves inside a bubble; a ranged intent counts as the 6.4 lock-in.
- **Class defaults only** — no inventory/equip/drops (deferred; see the `gear-equipment-system` note). Unarmed fallback (fists) baked in.
- Level scaling via ONE helper (`statsFor`/`weaponDamage`) so HP/damage/wire agree (like 6b.1's `levelFor`). Numbers are tunable `internal/protocol` constants.
- Backward-compatible join (default `ClassFighter` when unspecified) so existing join/e2e still work.
- Ranged attacks respect 6.4 domain scoping (target in the same bubble/range). `got, want`; `make check` + `make e2e` green. One PR off `main` (branch `m6b.2-classes`).

## Task 2: Game — class, per-class stats, level scaling

**Files:** `internal/game/world.go` (+ a new `internal/game/class.go` for the weapon/stats helpers); tests `internal/game/class_test.go`.

- [ ] `entity` gains `class string`. `class.go`: define the weapon type and class-default loadouts, and the stat helpers:

```go
type weapon struct {
	damage    int // base (level-1) damage
	rangeHex  int // 0 = melee/adjacent
	aoeRadius int // 0 = single target
}

// closeWeapon/rangedWeapon return the class default; a nil ranged weapon means
// the class has no ranged attack (fighter). Unarmed fallback (fists) applies
// when a class has no close weapon (forward-compat with gear).
func closeWeapon(class string) weapon { /* fighter=sword, rogue=dagger, mage=staffBonk, default=fists */ }
func rangedWeapon(class string) (weapon, bool) { /* rogue=bow, mage=staffMagic, fighter=none */ }

func maxHPFor(class string, level int) int  { return baseMaxHP(class) + protocol.HPPerLevel*(level-1) }
func weaponDamage(w weapon, level int) int  { return w.damage + protocol.DamagePerLevel*(level-1) }
```

- [ ] `Join(token, class)` (thread the class through; default `ClassFighter` for empty/unknown). A joined player's `maxHP`/`hp` come from `maxHPFor(class, 1)`.
- [ ] **Level scaling of HP**: on level-up / respawn, recompute `maxHP = maxHPFor(class, levelFor(xp))` (heal to new max on level-up; respawn already resets to maxHP). Keep the current-HP semantics sane (don't shrink below current on scale-up).
- [ ] `Snapshot` sets `Class: e.class`. (`MaxHP` already on the wire reflects the class/level max.)
- [ ] **Tests**: fighter has more MaxHP than rogue/mage at level 1; MaxHP grows with level (`maxHPFor` monotonic); a joined class-X player snapshots `Class==X` and the right MaxHP; default class when unspecified.
- [ ] Verify + `make lint`; commit — `"game: classes, per-class stats, level scaling"`.

## Task 4: Game — ranged attack intent (bow single-target, mage AoE)

**Files:** `internal/game/world.go`; tests `internal/game/ranged_test.go`.

- [ ] **Intent routing**: `SubmitIntent` handles `req.Kind`:
  - `"move"` (default): as today (pathfind, set path).
  - `"attack"`: only if the entity has a `rangedWeapon` and `Target` is within its `rangeHex` (and, for bow, a hostile is at `Target`; mage AoE allows any in-range hex). Record a pending ranged attack on the entity (e.g. `e.attackTarget *protocol.Hex`) and clear its path. If in a bubble, this counts as the **lock-in** (reuse the 6.4 all-ready trigger).
  - Reject (`Err... `) if the class has no ranged weapon (fighter), or out of range.
- [ ] **Resolution** in the attack phase (`attackLocked` / the combat pipeline, scoped to the bubble members): for each entity with a pending ranged attack:
  - **Bow** (`aoeRadius==0`): damage the opposing-faction occupant(s) at `Target` — a stack → a random member (seeded, reuse the 6.3 victim pick). Only if a hostile is there and within range.
  - **Magic** (`aoeRadius>0`): damage **every opposing-faction** entity within `aoeRadius` of `Target` (the target hex + ring) that is within `MageRange`. **No friendly fire** (skip same-faction).
  - Damage = `weaponDamage(rangedWeapon, levelFor(xp))`, accumulated against pre-attack HP with the bump damage (simultaneous). Clear `attackTarget` after resolution.
- [ ] Ranged attacks only apply to targets in the **same bubble/domain** (the member set passed to the combat pipeline) — a ranged intent whose target isn't in-range/in-domain does nothing (or is rejected at submit).
- [ ] **Tests** (`ranged_test.go`, pinned seeds, bridges): a rogue at distance 3 (≤ `BowRange`) from a monster in one bubble ranged-attacks → monster HP drops by bow damage, rogue doesn't move; out of `BowRange` → rejected/no damage; a mage AoE at a hex with 2 adjacent monsters → **both** take `StaffMagicDamage`; a friendly player in the AoE is **unharmed**; a fighter submitting an attack intent is **rejected**; a ranged intent in a bubble counts as lock-in (bubble resolves). 
- [ ] Verify + `make lint`; commit — `"game: ranged attack intents — bow single-target + mage AoE"`.

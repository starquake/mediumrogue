# Milestone 6b.3 — Species Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or executing-plans. Steps use `- [ ]` checkboxes.

**Goal:** Three species passives — human +XP, elf crit, dwarf damage-reduction — chosen at join alongside class.

**Architecture:** `Entity.Species`; two DRY helpers (`critMultiplier(attacker,rng)`, `applyDR(victim,dmg)`) fold into the three existing damage sites (melee bump, bow, AoE); the human XP bonus multiplies the bubble kill-XP award. All numbers are tunable `internal/protocol` constants.

## Global Constraints
- Species is **required** at join (no backward-compat default — see the `no-backward-compatibility` rule; client always sends it, server rejects empty/invalid with `ErrInvalidSpecies`). Species is player-only; monsters have none (`critMultiplier`/`applyDR` no-op for non-player-species).
- **Determinism**: the elf crit roll uses the EXISTING seeded per-turn RNG, rolled once per attack in the existing attack order; no elves → no rolls → existing seeded tests unchanged (same as the 6b.2 ranged pass).
- Crit then DR per hit: `applyDR(victim, base*critMultiplier(attacker,rng))`. DR floors a hit at 1.

```go
// critMultiplier rolls one crit for the attacker: an elf crits with
// ElfCritChancePercent chance for ElfCritMultiplier× damage; everyone else 1.
// Consumes rng only for an elf (so non-elf combat is RNG-identical to before).
func critMultiplier(attacker *entity, rng *mrand.Rand) int {
	if attacker.species == protocol.SpeciesElf && rng.IntN(100) < protocol.ElfCritChancePercent {
		return protocol.ElfCritMultiplier
	}
	return 1
}

// applyDR reduces a hit dealt to a dwarf by DwarfDamageReduction, floored at 1
// (a hit always lands for something). Others take full damage.
func applyDR(victim *entity, dmg int) int {
	if victim.species == protocol.SpeciesDwarf {
		if r := dmg - protocol.DwarfDamageReduction; r > 1 {
			return r
		}
		return 1
	}
	return dmg
}
```

- [ ] **Wire into the 3 damage sites** in `world.go` (the base damage stays `attackDamage`/`weaponDamage` as today; wrap the `damage[...] +=`):
  - **Melee bump** (`attackLocked`, ~`damage[victim.id] += attackDamage(a.attacker)`): `base := attackDamage(a.attacker); crit := critMultiplier(a.attacker, rng); damage[victim.id] += applyDR(victim, base*crit)`.
  - **Bow** (`resolveBowLocked`, single target): same shape — `crit := critMultiplier(e, rng); damage[victim.id] += applyDR(victim, dmg*crit)`.
  - **Mage AoE** (`resolveAoELocked`): roll crit **ONCE before the target loop** (`crit := critMultiplier(e, rng)`), then per target `damage[o.id] += applyDR(o, dmg*crit)` (a crit cast crits its whole splash).
  (Ensure each site has `rng` in scope — `attackLocked`/`resolveBow`/`resolveAoE` already receive it. `a.attacker`/`e` is the attacker entity.)
- [ ] **Human XP** (`resolveBubbleTurnLocked`, the `e.xp += killed * protocol.MonsterXP` per surviving player): multiply for humans — `award := killed * protocol.MonsterXP; if e.species == protocol.SpeciesHuman { award = award * (100 + protocol.HumanXPBonusPercent) / 100 }; e.xp += award`. (Or a `speciesXP(species, base) int` helper.)
- [ ] **Tests** (`species_test.go`, pinned seeds, `SetSpeciesForTest` bridge): human gains 1.5× MonsterXP on a kill vs a non-human (both surviving one fight); an elf with a crit-forcing seed deals `ElfCritMultiplier×` a bump AND a bow hit, and with a miss seed deals base (both branches — proves the roll); a dwarf takes `hit-DwarfDamageReduction` (floored ≥1) from a bump and a ranged hit; elf-hits-dwarf applies crit THEN DR; a non-elf/dwarf/human baseline is unchanged. Do NOT weaken; pin seeds so crit outcomes are deterministic.

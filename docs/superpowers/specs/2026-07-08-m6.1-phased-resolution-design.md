# Milestone 6.1 ‚Äî Phased (move) resolution: Design

*Status: approved 2026-07-08. Second slice of the milestone-6 decomposition
(after the 6.0 heartbeat warmup). Retires the milestone-3 ascending-entity-ID
move-resolution placeholder.*

## Goal

Replace the placeholder `resolveTurn` (ascending-entity-ID order with a per-move
occupancy re-check) with the **decided phased resolution's move phase**: all
moves resolve *simultaneously*, and hex overflow past `StackCap` is settled by a
**server-seeded, reproducible RNG tie-break** ‚Äî no ascending-ID favoritism. Pure
`internal/game` logic, testable with players now.

This is the **move phase only**. The *attack phase* (bump-to-attack, damage,
attacks against post-move positions) is milestone 6.3 ‚Äî there are no
hostiles or attacks yet. One hex per turn stays; multi-hex travel is still
deferred.

## The algorithm (invariant-preserving, seeded)

Each entity is a **mover** (non-empty `path` ‚Üí `target = path[0]`) or a
**stayer** (`target = current`). Resolution, holding `w.mu`:

1. **Seed the board.** `occ[hex]` starts at the current occupancy ‚Äî every entity
   reserves its own slot.
2. **Collect movers** in a canonical order (sort by id), then **shuffle** them
   with a per-turn seeded RNG (see below). The shuffle order is the tie-break.
3. **Admit in shuffle order.** For each mover with `target`:
   - if `occ[target] < StackCap`: admit ‚Äî `occ[current]--`, `occ[target]++`,
     move the entity, pop the path step.
   - else: **wait** ‚Äî the entity stays this turn, its path is retained for next.
4. `turn++`.

Properties this guarantees:
- **`occ[hex]` never exceeds `StackCap`** (only incremented when `< StackCap`) ‚Äî
  the invariant the conflict tests assert.
- **Vacating frees a slot**: an entity leaving a hex decrements it, so an
  incoming mover processed afterward can take the freed slot.
- **No favoritism**: overflow ties break by the seeded shuffle, not entity id.
- **Deterministic & reproducible** for a given (seed, turn, board).

### Accepted simplification

This is a single seeded pass, not a full simultaneous fixpoint. In a rare
full-hex "vacate-and-fill" swap, a mover processed *before* the same-turn
vacater can be rejected even though the slot ultimately frees. It **never
overflows** `StackCap`, and the case is a low-stakes edge at this player count ‚Äî
documented, not solved. (A future fixpoint pass could tighten it if playtests
show it mattering.)

## Seed handling (decided: random world seed ‚äï turn)

- `World` gains a `seed int64`, minted at construction from `crypto/rand` (a
  failed read leaves a zero seed ‚Äî still valid, just less random).
- Per turn the shuffle RNG is seeded from `w.seed ^ w.turn` (via
  `math/rand/v2`). Reproducible given the world seed + turn, but unpredictable to
  players (they don't know the world seed).
- A test-only bridge `SetSeedForTest(seed int64)` (in `export_test.go`) lets
  tests pin the seed and assert exact tie-break outcomes.
- The tie-break RNG is not cryptographic; if `gosec` (G404) flags `math/rand`,
  suppress with a `//nolint:gosec` carrying the reason (deterministic game
  tie-break, reproducibility required) ‚Äî per the repo's nolint-with-reason rule.

## Tests

**Invariants still hold (existing, unchanged):** `TestFriendlyStackingConverges`
and `TestStackCapBlocksOverflow` (in `world_test.go`) assert observable
invariants, not the tie-break winner, and must keep passing under the new
algorithm.

**New (`world_test.go`):**
- **Reproducibility:** two fresh worlds with the *same* pinned seed and the same
  6-movers-into-one-hex overflow resolve to the *same* blocked entity.
- **No ascending-id favoritism:** across a handful of pinned seeds, the blocked
  entity in a 6-into-one overflow is *not always the max id* ‚Äî proving the
  seeded shuffle replaced the old "highest id always loses" placeholder. (At
  least one seed blocks a non-max entity; deterministic, not flaky.)
- **Overflow invariant under the new path:** 6 entities all *move* into one empty
  hex ‚Üí exactly `StackCap` land there and exactly one is blocked (stays, path
  retained).

## Out of scope (later slices)

- Attack phase / bump-to-attack / damage ‚Äî 6.3.
- Monsters/hostiles ‚Äî 6.2.
- Time bubbles ‚Äî 6.4.
- Multi-hex-per-turn travel ‚Äî still deferred (plan ¬ß5 "maybe").

## Risks

- **`math/rand` vs `crypto/rand` import clash:** `world.go` already imports
  `crypto/rand` as `rand` (token generation). Add `math/rand/v2` under an alias
  (e.g. `mrand`); do not shadow the existing `rand`.
- **Reproducibility test brittleness:** assert *same seed ‚Üí same result* and
  *not-always-max-id*, not a hardcoded PCG output ‚Äî so the tests survive a stdlib
  RNG-stream change.

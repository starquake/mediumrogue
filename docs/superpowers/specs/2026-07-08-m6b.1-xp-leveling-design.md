# Milestone 6b.1 — XP & Leveling: Design

*Status: DRAFT (2026-07-08). First slice of milestone 6b (classes/species), and
it completes a milestone-6 debt: 6.3 shipped death with "**no XP penalty yet —
that's 6b**". This slice adds XP, levels, shared XP-on-kill, and the decided
XP-level death penalty. Builds on 6.3 (combat/death) + 6.4 (bubbles). Branch off
`main` (milestone 6 is merged).*

## Goal

Characters earn **XP** from kills and gain **levels**; death has a real but
bounded sting. Per the plan's decided rules:
- **XP from slain enemies, shared generously**: the moment a monster falls,
  **everyone in that fight gets the same, full amount** — no last-hit
  competition; running over to help always pays.
- **Levels are permanent**; XP is what you gamble.
- **Death penalty**: you **fall back to the start of the XP level you were in** —
  losing the progress inside that level, but never the level itself. ("I'm 80% to
  level 6" is exactly what a bad fight costs you.)

## Model

- Each **player** entity gains an `xp int` (cumulative, floored at level starts on
  death). **Level is derived** from XP: `level = 1 + xp/XPPerLevel` (1-based for
  display). Monsters have no XP (fields stay 0).
- **XP on kill**: when a monster dies inside a combat bubble (all combat happens
  in bubbles — you must be adjacent to attack, which is within `CombatRadius`),
  **every player member of that bubble** gains `MonsterXP` (the full amount each,
  not split). Implemented where deaths resolve in the bubble turn
  (`resolveDeathsLocked` / the bubble resolution), so it's the "everyone standing
  in the fight" set.
- **Death penalty**: on player death (respawn), XP falls to the **current level's
  floor**: `xp = (xp / XPPerLevel) * XPPerLevel`. Level is unchanged; within-level
  progress is lost. (Respawn already resets HP/position/path with the same
  id+token — 6.3; this adds the XP floor.)

## Constants (`internal/protocol`, tunable — gameplay feel)
- **`XPPerLevel = 100`** — XP needed per level (flat curve for now; a ramping
  curve is a later tuning call).
- **`MonsterXP = 20`** — XP awarded per monster kill (→ 5 kills per level).

## Wire (`internal/protocol`)
- `Entity` gains `XP int` (json `xp`) and `Level int` (json `level`). Server is
  authoritative for both (level derived server-side). Monsters send `0`/`1`.

## Client (`client/`)
- A small **stats HUD**: my **level** and an XP progress indicator (e.g. "Lv 6 ·
  80%" or a bar) from my entity's `xp`/`level` and `XPPerLevel`. `window.game`
  exposes `xp` and `level` for e2e.
- (Optional, low-cost) a brief "+XP" feedback on kill — deferred to the feel pass
  unless trivial.

## Decisions (reasoned calls; flag if you disagree)
1. **Flat XP curve** (`XPPerLevel=100`), **`MonsterXP=20`** — both tunable
   constants; picked for a simple first pass.
2. **Level is derived from XP** (not stored independently) — one source of truth;
   `level = 1 + xp/XPPerLevel`.
3. **"The fight" = the combat bubble**: XP goes to the killed monster's bubble's
   **player** members. Since all kills happen in bubbles, this is well-defined.
4. **Full XP to each participant** (shared, not divided) — matches the design's
   "same, full amount".
5. **Death floors XP to the level start** (keeps the level).

## Out of scope (later 6b slices / deferred)
- **Classes** (rogue/fighter/mage combat identities) — 6b.2.
- **Species** (human learns faster, elf crits, dwarf soak) — 6b.3. *Note: human's
  "learns faster" will scale XP gain — the hook lives here (a per-entity XP
  multiplier) but defaults to 1 until 6b.3.*
- **Gear/loot**, XP for non-combat (quests), XP scaling by monster type — later.
- **Level-up effects** (stat increases) — arrive with classes/species (6b.2/6b.3);
  this slice tracks level but a level grants no mechanical bonus yet.

## Tests
- **Unit (`internal/game`)**: killing a monster in a bubble grants `MonsterXP` to
  the killer; **all** player members of a bubble get the full `MonsterXP` on a
  kill (place 2 players + a low-HP monster in one bubble, resolve the kill, assert
  both gained `MonsterXP`); crossing `XPPerLevel` raises `Level`; **death floors
  XP** to the level start (give a player XP mid-level, kill them, assert XP ==
  level floor and Level unchanged); a fresh player is Level 1 / 0 XP.
- **Integration (`test/integration`)**: over HTTP, a player who kills a monster
  sees its `xp` rise in the bundle; on death `xp` drops to the level floor.
- **e2e**: `window.game.xp`/`level` reflect a kill; the stats HUD shows the level.

## Risks
- **Where the kill is credited**: award XP at the moment a monster's HP hits ≤0
  during a bubble resolution, to the bubble's *current* player members (before the
  bubble possibly dissolves that same turn). Get the ordering right (award before
  `resolveDeaths` removes the monster / recompute dissolves the bubble).
- **Death-floor arithmetic** off-by-one: define `level`/`floor` once and reuse
  (a `levelFor(xp)` helper) so the HUD, the penalty, and the wire agree.

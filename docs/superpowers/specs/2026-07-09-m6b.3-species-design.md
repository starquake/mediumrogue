# Milestone 6b.3 — Species: Design

*Status: DRAFT (2026-07-09). Third slice of 6b; the species passives that pair
with the 6b.2 classes. Builds on 6b.1 (XP), 6b.2 (classes/combat). Branch off
`main`.*

## Goal

Three species with a single passive each (plan §0, decided):
- **Human** — *learns faster*: **+XP** (a bonus on XP earned).
- **Elf** — *lands critical hits more often*: a chance for an attack to **crit**
  (extra damage).
- **Dwarf** — *shrugs off part of every hit*: **damage reduction** on incoming hits.

Species is chosen at join **alongside class** (a fighter-dwarf, a mage-elf, …).

## The passives (numbers tunable — flagged)

| Species | Passive | Default |
|---|---|---|
| Human | XP earned ×(1 + bonus) | `HumanXPBonusPercent = 50` → 1.5× XP |
| Elf | per-attack crit chance → damage ×mult | `ElfCritChancePercent = 20`, `ElfCritMultiplier = 2` |
| Dwarf | each incoming hit reduced (floored ≥1) | `DwarfDamageReduction = 1` (a hit still does ≥1) |

Where each hooks into the existing combat (6.3/6.4/6b.2):
- **Human XP** — in the bubble kill-XP award (`resolveBubbleTurnLocked`, where a
  surviving player gains `killed * MonsterXP`): multiply that award by the human
  bonus for human players.
- **Elf crit** — when an attacker deals damage, roll one crit per attack with the
  **existing seeded per-turn RNG** (so it's reproducible and doesn't perturb
  non-elf tests — no elves → no roll). Applies to all three damage paths (melee
  bump, rogue bow, mage AoE — one roll per cast; a crit AoE crits all its targets).
- **Dwarf DR** — when damage is dealt **to** a dwarf, subtract `DwarfDamageReduction`
  (floor the hit at 1, so DR softens but never fully negates). Applies at every
  damage site.

Two small helpers keep it DRY, used at the melee/bow/AoE sites:
`critMultiplier(attacker, rng) int` (elf → maybe `ElfCritMultiplier`, else 1) and
`applyDR(victim, dmg) int` (dwarf → `max(1, dmg-DwarfDamageReduction)`, else dmg).
Order per hit: **crit (attacker) then DR (victim)** — `applyDR(victim, base*crit)`.

## Wire (`internal/protocol`)
- `Species` constants (`SpeciesHuman`/`SpeciesElf`/`SpeciesDwarf`) + the passive
  constants above.
- `Entity.Species string` (json `species`); `JoinRequest.Species string` (json
  `species`). **Required** (no backward-compat default — the client always sends
  it; the server rejects an invalid/empty species on a new join, like class).

## Client (`client/`)
- **Species picker at join** next to the class picker: three buttons
  (Human/Elf/Dwarf); send `species` in the join; a default pre-selection (human)
  so existing e2e still auto-join. Persist with identity.
- `window.game.species` (my species) for e2e.
- Rendering: no per-species visual needed (keep light); the class glyph stays.

## Decisions (reasoned; flag if you disagree)
1. **Numbers** (all tunable constants): Human +50% XP, Elf 20% crit / 2× dmg,
   Dwarf −1 flat DR (min 1 per hit). Simple first pass; balance later.
2. **Crit = one roll per attack** (not per AoE target) — a crit cast crits its
   whole splash; deterministic via the existing seeded RNG.
3. **DR floors a hit at 1** (never fully negates) so combat always progresses.
4. **Species chosen at join, immutable** (like class) — no respec yet.

## Out of scope (later)
- **Gear/weapons variety** (the `gear-equipment-system` note); **respec**;
  **more/finer species traits**; **crit on monsters** (monsters have no species —
  they don't crit or reduce). Species passives are player-only.

## Tests
- **Unit (`internal/game`)**: a **human** gains 1.5× `MonsterXP` on a kill vs a
  non-human's `MonsterXP` (same fight, both surviving). An **elf** attacker with a
  pinned seed that forces a crit deals `ElfCritMultiplier×` weapon damage; with a
  seed that misses, base damage (proves the roll is real, both branches). A
  **dwarf** victim takes `hit − DwarfDamageReduction` (floored ≥1) from a bump AND
  a ranged hit. Crit + DR together (elf hits a dwarf) applies crit then DR. A
  non-elf/non-dwarf/non-human is unaffected (baseline unchanged → existing tests
  pass). Pin seeds; use `SetSpeciesForTest` bridge.
- **Integration**: join as each species → `Species` on the wire; a human’s XP
  rises faster on a kill (compare to a non-human). (Crit is RNG — assert a trend
  or use the unit tests for exactness.)
- **e2e**: pick a species → `window.game.species`; the picker + class picker both
  work and still auto-join.

## Risks
- **Determinism**: the elf crit roll consumes the seeded RNG; roll it in a stable
  order (per attack, in the existing sorted/shuffled attack order) so a full turn
  stays reproducible. No elves → no rolls → existing seeded tests byte-identical
  (same discipline as the 6b.2 ranged pass).
- **Two pickers at join**: class + species must both send valid required values
  and still auto-join with defaults (fighter + human) so existing e2e don't hang.
- **DR/crit only for players**: monsters have no species — `critMultiplier`/
  `applyDR` no-op for them (attacker/victim not a player-species). Keep monster
  combat unchanged.

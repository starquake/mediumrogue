# medium-rogue

A multiplayer roguelike for a ~15-friend group: shared hex world, simultaneous
5-second turns (WeGo), local combat time bubbles, browser client. The full
design — every decided rule and open question — lives in
**`docs/roguelike-mp-plan.md`**; read it before making design-level choices.

**Resuming work? Start with `docs/STATUS.md`** — current milestone state,
next step, known placeholders, and environment gotchas. Keep it updated at
the end of every session.

## Architecture

- **Go server** (module root, standard go.dev server layout): authoritative
  simulation. `cmd/rogue` → `cmd/rogue/app` (lifecycle) → `internal/server`
  (HTTP/SSE) → `internal/game` (world clock, later: turn resolution).
- **TypeScript + PixiJS client** (`client/`, Vite): renderer + intent sender.
  Built bundle is embedded into the Go binary via `internal/web` (go:embed).
- **Wire**: HTTP POST up (intents), SSE down (`/api/events`: turn bundles,
  heartbeat comments). JSON payloads. SSE ids are turn numbers
  (Last-Event-ID resume).
- **Protocol**: `internal/protocol` is the single source of truth.
  `client/src/protocol.gen.ts` is GENERATED by tygo — never edit it by hand;
  run `make protocol` after changing protocol types.

## Commands

- `make check` — full pre-commit gate (lint, protocol drift, typecheck, tests, build)
- `make dev` + `make client-dev` — two-terminal dev loop (Go auto-restart via watchexec; Vite HMR proxying /api to :8080)
- `make test` / `make test-integration` — Go unit / real-HTTP integration tests
- `make e2e` — Playwright against the real embedded-client binary (needs `npx playwright install chromium` once)
- `make lint-fix` — golangci-lint with autofix
- `make smoke` — config + wiring check without serving

Go may not be on PATH; the Makefile falls back to `/usr/local/go/bin/go`.

## Conventions

- Patterns are adapted from `starquake/topbanana` (Makefile chassis, server
  layering, coalescing hub, integration-test style). When adding machinery,
  check how topbanana does it first.
- The hub (`internal/hub`) coalesces: a tick means "fetch the latest state",
  never a delta. Don't build logic that assumes one tick per event.
- Tests: unit tests live next to code; `test/integration` drives the real
  handler tree over real HTTP; `client/e2e` drives the real binary in a real
  browser. The client exposes `window.game` for Playwright — keep it in sync
  when adding client state.
- Timing knobs (`TURN_INTERVAL`, `HEARTBEAT_INTERVAL`) are env-configurable
  precisely so tests can shrink them — thread new intervals the same way.
- Game-rule constants (turn cadence, combat radius, stack cap) live in
  `internal/protocol` so both sides compile against the same numbers.

## Maintenance reminders

- **Periodically audit all `//nolint` suppressions**
  (`grep -rn '//nolint' cmd internal test`): re-justify or remove each. Every
  suppression must carry a reason comment; if the reason no longer holds (or
  a disabled linter in `.golangci.yml` starts fitting the codebase), fix the
  code instead. Also revisit the `disable:` list and test-file exclusions in
  `.golangci.yml` — they were calibrated at milestone 1–2 scale.
